This system facilitates secure, state-\/driven communication using a custom-\/defined network protocol. The Flight Tower manages airspace safety via a robust state machine, while the Pilot application reports telemetry and requests vital flight documentation. \hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{Key Features}{Key Features}}\label{md__r_e_a_d_m_e_autotoc_md1}

\begin{DoxyItemize}
\item Custom Packet Protocol:\+ Implements a strict Head \texorpdfstring{$\vert$}{|} Body \texorpdfstring{$\vert$}{|} Tail structure with bit-\/field headers.
\item State-\/\+Driven Logic:\+ Server functionality is restricted based on operational states (e.\+g., Idle, Active, Emergency).
\item Large Data Transfer:\+ Reliable TCP-\/based transfer of a 1MB+ Flight Manual.
\item Black Box Logging:\+ Automated persistent logging of all TX/\+\+RX network traffic for safety audits.

The Flight Tower operates through the following transitions
\begin{DoxyItemize}
\item IDLE:\+ Waiting for Pilot authentication.
\item PREFLIGHT:\+ Validated pilot; preparing for taxi/\+takeoff.
\item ACTIVE:\+ Monitoring live telemetry and traffic.
\item DATA\+\_\+\+TRANSFER:\+ Dedicated state for 1MB manual transmission.
\item EMERGENCY:\+ High-\/priority state triggered by "{}\+Mayday"{} commands.
