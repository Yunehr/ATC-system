Detailed Description

Assembles a multi-packet file transfer into a local file.

Each packet payload begins with a 4-byte little-endian write offset, followed by the file data for that chunk. This allows packets to arrive out of order without corrupting the output.

The file is first written to a .part staging path. finalize() flushes, closes, and atomically renames it to the final path, ensuring no truncated file is ever left at the destination on failure or interruption.

FileReceiver is owned and driven by ClientEngine::downloadFlightManual().

bool FileReceiver::processPacket ( packet & pak,
std::string & error )

Writes the payload of a single pkt_dat packet into the staging file.

Writes the payload of a single data packet into the staging file.

Reads the 4-byte write offset from the start of the payload, seeks to that position, and writes the remaining bytes. Updates bytesReceived to the high-water mark of all write positions.

Sets the receiver complete if the packet's transmission flag is PKT_TRNSMT_COMP.

Parameters
pakThe received packet. Must be of type pkt_dat.
errorPopulated with a human-readable message on failure.
Returns
true if the packet was written successfully.
false on type mismatch, invalid payload, or I/O error; error is set.
Packet payload layout
[ 4 bytes: uint32_t write offset ] [ N bytes: file data ]
The offset is applied via seekp() before writing, so packets may arrive out of order without corrupting the output.

bytesReceived is updated to track the highest byte position written, giving an accurate progress figure even with out-of-order packets.

Sets complete to true if the packet's transmission flag is PKT_TRNSMT_COMP, signalling that no further chunks will follow.

Parameters
pakThe received packet to process. Must be of type pkt_dat.
errorPopulated with a human-readable message on failure.
Returns
true if the packet was written successfully.
false on type mismatch, invalid payload size, seek failure, or write failure; error is set.

Opens the staging file and prepares the receiver for a new transfer.

The incoming file is written to <outputPath>.part to avoid leaving a truncated file at the final path if the transfer fails or is interrupted. finalize() atomically renames the staging file to outputPath on success.

If a staging file is already open from a previous session it is closed before the new one is opened.
