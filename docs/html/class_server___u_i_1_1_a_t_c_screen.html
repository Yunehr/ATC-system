Detailed Description

Main application class for the ATC Tower Control Center dashboard.

Responsibilities:

  • Launching and monitoring the C++ server subprocess.
  • Reading server stdout on a daemon thread and routing messages to the UI via thread-safe queues.
  • Polling netstat every second to display live TCP connection endpoints.
  • Updating metric cards, the system log, and the file transfer progress bar.
  • Cleanly terminating the server process on window close.

Parses a server log line for structured file-transfer progress events.

Recognises three prefixes emitted by the C++ server:

  • "TRANSFER START:total=<N>" — a new transfer beginning with N total bytes.
  • "TRANSFER PROGRESS:current=<C> total=<T>" — an in-progress update.
  • "TRANSFER COMPLETE:current=<C> total=<T>" — the transfer has finished.
Parameters

Returns

◆ read_server_output()

Server_UI.ATCScreen.read_server_output ( self)

Reads server stdout lines on a daemon thread and queues them for the UI.

For each line received:

  • Pushes the raw text to log_queue for display in the log panel.
  • Calls parse_transfer_event and, if a transfer event is found, schedules handle_transfer_event on the Tk main thread.
  • Detects "STATE:" prefixed lines and schedules set_system_state.
  • Detects connection/disconnection keywords and pushes events to connection_queue.

Exits when server_running is False or the stdout stream closes.

Read server output in a separate thread
