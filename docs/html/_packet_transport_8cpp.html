Detailed Description

Winsock send/receive helpers that frame packet objects over TCP.

Bridges the packet serialization layer and the raw Winsock socket API. All four functions guarantee that either the full requested byte count is transferred or false is returned — partial transfers are handled internally by the sendAll / recvAll loop pair.

Wire format assumed by receivePacket

[ header (2 bytes) | payload (0–249 bytes) | CRC (4 bytes) ]

The payload length is read from byte 1 of the header, matching the packet::header layout defined in packet.h.
