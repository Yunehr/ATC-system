Detailed Description

Per-session state machine enforcing the aviation client protocol.

Each StateMachine instance is owned by a single client session in ServerEngine::handleClient() and is not shared between connections.

Permitted requests per state are read from StatePermissions.csv at runtime via canHandle(), so the permission matrix can be updated without recompiling the server. One hard-coded exception applies: req_file is always permitted in ACTIVE_AIRSPACE regardless of the CSV contents.

The previousState member provides a single level of state history, used to restore the pre-transfer or pre-emergency state after onDataTransferComplete() or onEmergencyResolved() is called.

void StateMachine::onRequestHandled ( reqtyp requestType)

Updates state in response to a successfully completed request.

Updates the state in response to a successfully handled request.

Transition rules:

  • req_taxi from PRE_FLIGHT → ACTIVE_AIRSPACE (departure).
  • req_taxi from ACTIVE_AIRSPACE → PRE_FLIGHT (arrival).
  • req_file (any state) → saves current state to previousState and transitions to DATA_TRANSFER.
  • req_telemetry / req_traffic → forces ACTIVE_AIRSPACE.
  • All other types leave the state unchanged.
