Detailed Description

Declaration of the ServerEngine class.

ServerEngine owns the TCP listen socket and drives the full server lifecycle: binding, accepting, per-client packet dispatch, and shutdown. Each accepted connection is handled synchronously (one client at a time) by handleClient(), which owns its own StateMachine instance for fully isolated session state.

Structured stdout protocol
ServerEngine writes machine-readable prefixed lines to stdout that are consumed by the Python ATCScreen dashboard:
Prefix Purpose
"STATE: <name>" Server state transition.
"TRANSFER START: total=<N>" File transfer starting.
"TRANSFER PROGRESS: current=<C> total=<T>" Chunk sent.
"TRANSFER COMPLETE: current=<C> total=<T>" Transfer finished.
"Client connected!" New connection accepted.
"Client disconnected." Connection closed.
