Detailed Description

Implementation of the ServerEngine class.

Manages the TCP listen socket, accepts incoming client connections (one at a time, sequentially), and drives the per-client packet dispatch loop. Each client session owns its own StateMachine instance, so state is fully isolated between connections.

Structured stdout protocol
In addition to human-readable log lines, ServerEngine writes machine- readable prefixed lines consumed by the Python ATCScreen dashboard:
Prefix Meaning
"STATE: <name>" Server state changed; update UI indicator.
"TRANSFER START: total=<N>" File transfer beginning; N total bytes.
"TRANSFER PROGRESS: current=<C> total=<T>" Chunk sent.
"TRANSFER COMPLETE: current=<C> total=<T>" Transfer finished.
"Client connected!" New TCP connection accepted.
"Client disconnected." TCP connection closed.
