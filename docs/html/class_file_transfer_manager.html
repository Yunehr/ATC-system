Detailed Description

Static utility class for server-side data file access.

All methods are static; FileTransferManager is not intended to be instantiated. File paths are resolved at call time using a two-path probe (../data/ then data/) so the same binary works from both the project root and the build output directory.

Checks StatePermissions.csv to determine whether taxi or landing clearance should be granted in the given server state.

Checks StatePermissions.csv to determine whether a taxi or landing clearance should be granted in the current server state.

Grants clearance only if TAXI_REQUEST appears in the dash-delimited command list for currentState. The response string distinguishes between departure and arrival clearance so the Python UI can navigate to the correct page.

Parameters
currentStateThe server's current state string (e.g. "PRE_FLIGHT", "ACTIVE_AIRSPACE").
Returns
One of:
  • "Taxi Clearance: APPROVED for departure sequence"
  • "Landing Clearance: APPROVED for arrival sequence"
  • "Taxi Clearance: DENIED by state policy"
  • "Taxi Clearance: unable to verify permissions"

Reads the permissions CSV and searches for a row whose StateName column matches currentState. If the TAXI_REQUEST command is present in that row's dash-delimited command list, clearance is granted. The response string distinguishes between departure (state PRE_FLIGHT) and arrival (state ACTIVE_AIRSPACE) so the Python UI can navigate to the correct page.

Parameters
currentStateThe server's current state string (e.g. "PRE_FLIGHT", "ACTIVE_AIRSPACE").
Returns
One of:
  • "Taxi Clearance: APPROVED for departure sequence"
  • "Landing Clearance: APPROVED for arrival sequence"
  • "Taxi Clearance: DENIED by state policy"
  • "Taxi Clearance: unable to verify permissions" if the file could not be opened.

Appends a telemetry record to SystemTrafficLog.csv.

Seeds the next LogID from the highest ID already in the file, then extracts Lat and Long from telemetry and writes them as discrete columns. An empty telemetry string is recorded as "NO_TELEMETRY".

Parameters
telemetryRaw telemetry payload (e.g. "Lat=51.5,Long=-0.1,Alt=10000,Speed=450"), or empty to log a placeholder entry.
clientIdNumeric client identifier included in the confirmation message.
pilotIdPilot identifier written to the PilotID CSV column.
Returns
A confirmation string (e.g. "Telemetry logged for client 1 (LogID=42)"), or "Telemetry: failed to open SystemTrafficLog.csv" on failure.

Reads the current highest LogID from the file (if it exists) to ensure the new entry receives a unique, monotonically increasing ID. Extracts Lat and Long fields from telemetry via parseField() and writes them as discrete columns alongside the full telemetry string.

CSV row format written
LogID, PilotID, Lat, Long, Status, Telemetry, Timestamp

If telemetry is empty, the Telemetry column is written as "NO_TELEMETRY" to avoid a blank CSV field.

telemetryRaw telemetry string (e.g. "Lat=51.5,Long=-0.1,Alt=10000,Speed=450"), or empty to log a placeholder entry.
clientIdNumeric client ID, included in the confirmation message.
pilotIdPilot identifier written to the PilotID CSV column.
Returns
A confirmation string (e.g. "Telemetry logged for client 1 (LogID=42)"), or "Telemetry: failed to open SystemTrafficLog.csv" on failure.
