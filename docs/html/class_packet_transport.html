bool PacketTransport::receivePacket ( int sock,
packet & outpak )
static

Receives one complete packet from sock and reconstructs it.

Receives exactly one packet from a TCP socket into outpak.

Performs a two-phase read: first the fixed-size header to determine payload length, then the remaining body and CRC tail. The result is written into outpak via deep copy.

Parameters
sockConnected TCP socket descriptor.
outpakOutput parameter set to the received packet on success. Left unchanged on failure.
Returns
true on success; false if either read phase fails.
See also
recvAll()

Uses a two-phase read to avoid buffering the entire stream:

  1. Read the fixed 2-byte header to determine payload length.
  2. Allocate a full-packet buffer, copy the header in, then read the remaining body and CRC tail directly into it.

The completed buffer is passed to the packet(char*) constructor and the result is assigned to outpak via operator=, which performs a deep copy. The temporary buffer is freed before returning.

Parameters
sockConnected Winsock socket descriptor.
outpakOutput parameter populated with the received packet on success. Unchanged on failure.
Returns
true if a complete, well-formed packet was received; false if either recvAll() call fails (socket closed, connection reset, or other Winsock error).
Warning
The header size is hardcoded as 2 bytes to match the current packet::header bitfield layout. If that struct changes, this value must be updated to match. Consider exposing a packet::HEADER_SIZE constant to keep them in sync.
Payload length is read directly from the raw header byte with no validation against MAX_PKTSIZE. A malformed or malicious header could cause an oversized allocation.
