Detailed Description

Static utility class for CSV packet logging (REQ-SYS-050).

Static utility class for server-side CSV packet logging.

All methods are static; Logger is not intended to be instantiated. Internal state (log file path, entry counter, initialisation flag) is held in module-level statics in Logger.cpp and managed transparently by the private initialize() method, which is called lazily on the first logPacket() invocation.

Appends a single packet log entry to TransmitReceiveLog.csv.

Appends a single packet log entry to the CSV file.

Initialises the logger on first call (lazy). Sanitises description by replacing newline characters with spaces, then writes one CSV row in the format:

LogID, ClientID, Direction, PacketType, ServerState, Timestamp, Description

Silently returns without writing if the log file cannot be opened.

Parameters
pktThe packet to log; supplies the client ID and packet type.
dirTransmission direction: Direction::TX or Direction::RX.
serverStateThe server's current state string at the time of logging (e.g. "PRE_FLIGHT", "ACTIVE_AIRSPACE").
descriptionHuman-readable context string (e.g. "Auth request from client 1").

Calls initialize() on first use. The description string is sanitised by replacing newline characters with spaces before writing. The description is double-quoted in the output to allow commas within the text.

Unlike the client-side logger, this method also records serverState, providing a full audit trail of which server state was active when each packet was processed.

Output row format
<LogID>,<ClientID>,<RX|TX>,<PacketType>,<ServerState>,<YYYY-MM-DD HH:MM:SS>,"<description>"

Silently returns without writing if the log file cannot be opened for append.
