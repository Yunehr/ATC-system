Detailed Description

Manual usage and smoke tests for the packet and Request classes.

Author
Sebastian Solorzano

This file demonstrates and validates the core send/receive pipeline without a real network socket. In production, the only difference would be replacing the direct buffer hand-off with a send() / recv() pair over a TCP socket. TCP guarantees delivery integrity, so no additional corruption checks are needed at this layer beyond the CRC.

Three scenarios are exercised:

  1. Basic Request construction and serialization.
  2. Raw payload wrapped in a packet and reconstructed on the receive side.
  3. A serialized Request carried inside a pkt_req packet, then unwrapped.

int main ( void )

Smoke-tests Request and packet construction, serialization, and reconstruction without a live socket.

Test 1 — Request round-trip

Constructs a Request with an arbitrary type and body, serializes it, and prints the fields to confirm the values survive serialization.

Test 2 — Packet round-trip (raw payload)

Wraps a plain string in a packet using pkt_emgcy (chosen to exercise negative/high enum values). Serializes to a wire buffer, reconstructs a receive-side packet from that buffer, and prints all header fields plus the payload to verify round-trip fidelity.

Test 3 — Request carried inside a packet

Serializes a Request, stuffs it into a pkt_req packet, serializes the packet, reconstructs both the packet and the inner Request on the receive side, and prints all fields. This is the canonical pattern for real client messages.

Note
In real use, replace the direct buffer hand-off between Serialize() and the receiving packet constructor with send() / recv(). TCP ensures the bytes arrive intact, so no extra integrity step is needed beyond confirming the transmit_flag.
After receiving, read the packet's type flag with getPKType() and branch on the PKTYPE value — a few if statements are sufficient to dispatch to the correct handler.
