Detailed Description

An application-layer command consisting of a type byte and a body payload.

Owns its body buffer and manages its lifetime through RAII. Supports both the transmit path (construct from type + raw data → serialize) and the receive path (construct from a flat byte buffer → inspect fields).

Warning
serializeRequest() allocates a new buffer on every call. The caller is responsible for delete[]ing the returned pointer.
There is no copy constructor defined. If a Request is copied by value (e.g. passed to a handler by value), the default shallow copy will cause a double-free. Pass by reference or pointer, or add a copy constructor matching operator=.

char * Request::serializeRequest ( int32_t * finalsize)
inline

Serializes this Request into a flat byte buffer.

Allocates a new buffer of 1 + bodysize bytes, writes the type byte at index 0, then copies the body. The resulting layout matches what the receive-side constructor Request(char*, int32_t) expects.

Parameters
[out]finalsizeSet to the total number of bytes in the returned buffer (1 + bodysize). May be nullptr if the size is not needed.
Returns
Pointer to a heap-allocated serialized buffer. The caller is responsible for calling delete[] on this pointer.
Warning
Failing to delete[] the returned pointer is a memory leak.
