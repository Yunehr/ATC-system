Detailed Description

Entry point for the client API bridge process.

This executable acts as a thin stdin/stdout bridge between the Python UI (clientUI.py) and the aviation backend server. The UI spawns this process as a child, writes newline-terminated command strings to its stdin, and reads response strings from its stdout.

Command format (space-delimited, read from stdin):

<CMD> [arg1] [arg2]

Response format (newline-terminated, written to stdout):

<response string from ClientApp::apiRequest>

Argument handling
  • argv[1] : IPv4 address of the backend server. Defaults to "127.0.0.1" if not supplied.
  • argv[2] : TCP port number (1–65535). Defaults to 54000 if not supplied or out of range. Out-of-range values parsed by std::atoi (including 0 and negatives) are silently ignored and the default is kept.
