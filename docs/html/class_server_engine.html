Detailed Description

Manages the TCP listen socket and per-client packet dispatch loop.

Responsibilities:

  • Creating, binding, and listening on the server TCP socket (start()).
  • Accepting client connections sequentially in a blocking loop (run()).
  • Dispatching incoming packets to authentication, request, and emergency handlers within handleClient().
  • Streaming the flight manual PDF to a client in fixed-size chunks (sendFlightManual()).
  • Cleanly closing the listen socket on stop() or destruction.
Note
Winsock initialisation (WSAStartup / WSACleanup) is the caller's responsibility and must be performed before constructing this object.

Blocks in the accept loop, handling one client connection at a time.

Blocks in the accept loop, handling one client at a time.

For each accepted connection, calls handleClient() (blocking until the client disconnects), then closes the client socket before accepting the next one. Failed accept() calls are logged and skipped rather than treated as fatal. Exits when running is set to false by stop().

Calls accept() in a loop. For each accepted connection, calls handleClient() (which blocks until the client disconnects), then closes the client socket before accepting the next connection.

Invalid accept() results are logged and skipped rather than aborting the loop. The loop exits when running is set to false by stop().

bool ServerEngine::sendFlightManual ( SOCKET clientSock,
unsigned char clientId )

Streams the flight manual PDF to the client in fixed-size chunks.

Each pkt_dat payload is prefixed with a 4-byte little-endian write offset, matching the layout expected by FileReceiver::processPacket() on the client side. All chunks except the last carry PKT_TRNSMT_INCOMP; the final chunk carries PKT_TRNSMT_COMP.

Emits "TRANSFER START:", "TRANSFER PROGRESS:", and "TRANSFER COMPLETE:" lines to stdout for the ATCScreen dashboard.

If the file opens but contains zero bytes, a single zero-length pkt_dat packet with PKT_TRNSMT_COMP is sent so the client can exit its receive loop cleanly.

Parameters
clientSockThe connected client socket to write chunks to.
clientIdClient ID stamped into every outgoing packet header.
Returns
true if all chunks were sent successfully.
false if the file could not be opened or any send failed.
Note
This method is public to facilitate unit testing; it is normally called only from handleClient().
Chunk payload layout
Each pkt_dat packet payload contains:
[ 4 bytes: uint32_t write offset ] [ N bytes: file data ]
This matches the layout expected by FileReceiver::processPacket() on the client side.
Transmission flags
  • All chunks except the last carry PKT_TRNSMT_INCOMP.
  • The final chunk (detected via src.peek() == EOF) carries PKT_TRNSMT_COMP.
Progress reporting
Emits "TRANSFER START:", "TRANSFER PROGRESS:", and "TRANSFER COMPLETE:" lines to stdout for the ATCScreen dashboard.
Edge case — empty file
If the file opens successfully but contains zero bytes, a single zero-length pkt_dat packet with PKT_TRNSMT_COMP is sent so the client can exit its receive loop cleanly.

bool ServerEngine::start ( unsigned short port)

Binds a TCP listen socket to the given port and begins listening.

Creates an AF_INET/SOCK_STREAM socket, binds it to INADDR_ANY on port, and calls listen(). Emits "STATE: STARTUP/AUTH" to stdout on success so the ATCScreen dashboard can initialise its state indicator.

On any failure the socket is closed and reset to INVALID_SOCKET.

Returns
true if the socket is bound and listening.
