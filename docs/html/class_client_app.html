Detailed Description

Manages the Winsock connection and all aviation-domain request handlers.

Responsibilities:

  • Winsock initialisation and teardown (WSAStartup / WSACleanup).
  • Opening and closing the TCP connection via ClientEngine.
  • Dispatching commands received from the Python UI to the appropriate handler.
  • Tracking the current server-side state (REQ-CLT-040).

Constructs a ClientApp, initialises Winsock, and connects to the server.

Writes "CONNECTED\\n" to stdout on success, or "CONNECT_FAILED\\n" / "WSAStartup failed\\n" on failure. Check isConnected() before use.

Parameters
ipIPv4 address of the backend server. Defaults to "127.0.0.1".
portTCP port of the backend server. Defaults to 54000.

Performs WSAStartup and then attempts to open a TCP connection to the backend server. On success, writes "CONNECTED\\n" to stdout so that the parent process knows the bridge is ready. On any failure the corresponding flag (initialized / connected) is left false and subsequent calls to apiRequest() will return "ERROR_NOT_CONNECTED".

Parameters
ipIPv4 address of the backend server (e.g. "127.0.0.1").
portTCP port the backend server is listening on (e.g. 8080).

Dispatches a command string to the appropriate handler.

Parses and dispatches a command received from the stdin bridge.

Called once per line by the stdin bridge in main.cpp.

Parameters
cmdCommand token. Recognised values: EMERGENCY, RESOLVE, LOGIN, WEATHER, FLIGHT, TAXI, LAND, TELEM, TRAFFIC, MANUAL.
arg1First optional argument (e.g. username, airport code, flight ID).
arg2Second optional argument (e.g. password for LOGIN).
Returns
Response string to forward to the Python UI, or "ERROR_NOT_CONNECTED" if the socket is not ready, or "ERROR_UNKNOWN_CMD" if cmd is not recognised.

This is the main entry point called by main.cpp for every line read from stdin. It maps command tokens to handler methods and returns their response strings, which are then written back to stdout by the caller.

std::string ClientApp::handleTaxi ( )

Requests taxi or landing clearance from the server.

Transitions currentState to "ACTIVE_AIRSPACE" on a departure clearance, or back to "PRE_FLIGHT" on an arrival clearance.

Returns
Clearance response string from the server.

Inspects the response to determine the resulting state transition:

  • A "departure" keyword advances currentState to "ACTIVE_AIRSPACE".
  • An "arrival" keyword returns currentState to "PRE_FLIGHT".
Returns
The taxi/landing clearance response string from the server.

◆ handleTelemetry()

std::string ClientApp::handleTelemetry ( const std::string & data = "")

Sends a telemetry update to the server.

If data is non-empty it is forwarded directly. Otherwise, simulated GPS telemetry is generated with randomised values (lat, long, alt, speed).

Parameters
dataPre-formatted telemetry string, or empty to use the simulator.
Returns
Server acknowledgement response string.

If data is non-empty it is forwarded directly. Otherwise, simulated GPS telemetry is generated with randomised values in realistic ranges:

  • Latitude: 43.0 – 60.0 °N
  • Longitude: 140.0 – 60.0 °W
  • Altitude: 5,000 – 35,000 ft
  • Speed: 200 – 500 kt

The simulated payload is formatted as:

Lat=<val>,Long=<val>,Alt=<val>,Speed=<val>
Parameters
dataOptional pre-formatted telemetry string. Pass an empty string to use the built-in GPS simulator.
Returns
The server's acknowledgement response string.

◆ runCommandLoop()

void ClientApp::runCommandLoop ( )

Runs the legacy interactive stdin command loop.

Runs an interactive stdin command loop (legacy interface).

Reads whitespace-delimited tokens from stdin and dispatches them to the appropriate handler, printing each response to stdout. Exits on "QUIT" or stdin EOF.

Note
This loop is retained for debugging. The production entry point is apiRequest(), driven by the Python bridge in main.cpp.

Reads whitespace-delimited tokens from stdin and dispatches them to the appropriate handler. This loop is retained for debugging purposes; the production code path uses apiRequest() instead.

Recognised commands:

Command Action
QUIT Prints "BYE" and exits the loop.
EMERGENCY Calls handleEmergency().
Login Prompts for username/password interactively.
WEATHER Prompts for a location name.
FLIGHT Prompts for a flight ID.
TAXI Calls handleTaxi().
TELEM Calls handleTelemetry().
TRAFFIC Calls handleTraffic().
MANUAL Calls handleManualDownload().

Does nothing if the app is not initialised and connected.
